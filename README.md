# BarcodeReaderSystem

BarcodeReaderSystem は臨床検査技師がISO15189の要求事項に基づき、バーコードリーダーを使用して試薬類の使用開始、使用期限、ロット番号をwebアプリケーション上で管理するためのシステムです。
紙台帳や Excel による試薬管理は、ISO 15189 が求める記録（使用開始・期限・ロットのトレーサビリティ）を維持するうえで手間と転記ミス発生のリスクがあります。本システムは、GS1 コードを読み取り、試薬情報を構造化して記録することで、この課題を解決することを目的とします。

このリポジトリでは、まず最小構成で動作するもの（MVP）を制作します。

- アプリケーション
- GitHub

## 機能

- GS1コードの読み取り
  - カメラでGS1コードを読み取り、商品情報、使用期限、ロット番号を取得
- 試薬管理
  - コードから取得した情報をもとに、試薬の使用開始、使用期限、ロット番号をデータベースに記録

## 開発環境（フロントエンド）

- Next.js

## 開発環境（バックエンド）

- Next.js の API Routes をバックエンドとして使用する。
  ※ MVP 完成を目的とするため、専用のバックエンドサーバーは立てず、API Routes 内で処理を完結させる。

## 開発環境（データベース）

- Supabase（PostgreSQL）
  選定理由: MVP では Supabase を実質ホスティングされた PostgreSQL として用いるが、v1 で必要となる認証・RLS（行レベルセキュリティ）・複数ユーザー対応を Supabase の機能でそのまま拡充できることを見込んでの選定である。v1 への地続き性を技術選定の根拠とする。

## 画面

1. スキャン画面  
   GS1読み取り → 商品マスタ照合
   - 既存GTIN: ロット登録(gtin+lot_numberでユニーク、初回登録日=使用開始日、2回目以降は弾く)
   - 未登録GTIN: エラー表示
2. 商品登録画面 試薬名・メーカー・カテゴリを手入力(別UI、能動導線のみ)
3. 商品一覧画面 登録済み商品の一覧
4. ロット一覧画面 商品ごとのロット一覧(商品一覧からドリルダウン、使用開始日、使用期限、ロット番号、開封者を表示)

## データベース設計

※ v1前提で設計、MVPでは一部のみ運用

- products
  - gtin(PK), name, manufacturer, category
- reagent_lots
  - id, gtin(FK), lot_number, expiry_date, status, opened_at, created_at
    ※ (gtin, lot_number)にユニーク制約
- usage_records
  - id, lot_id(FK), used_at, user_id  
    ※列は作る、MVPでは最小運用
